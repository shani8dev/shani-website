Features & Architecture

Btrfs Dual-Root Architecture

Two independent roots (blue & green) ensure atomic, reliable upgrades.

Shanios uses a dual-root setup on Btrfs. While you‚Äôre running on one root, the other is updated in the background. This guarantees upgrades are fully validated before switching and provides an instant rollback option if needed.

Read-Only Root & Overlays

Immutable base plus writable overlays for configuration and runtime data.

Shanios is a read-only Arch Linux pacstrap image. Key directories like /etc are handled with overlays, and /var uses tmpfs, ensuring no unauthorized writes can compromise system integrity.

Atomic & Blue/Green Updates

Zero downtime upgrades with instant rollback capabilities.

Thanks to the dual-root approach, Shanios performs updates on the inactive slot. If the new slot boots successfully, you continue seamlessly. If not, you revert to the old slot‚Äîensuring system stability with no partial upgrades.

Minimal GNOME Environment

A sleek GNOME desktop with minimal overhead or bloat, OEM-ready.

After installation, Shanios defaults to GNOME with gnome-initial-setup‚Äîperfect for OEM use or personal customization. You get a modern desktop that‚Äôs minimal yet functional, ideal for both newcomers and power users.

Sway-Powered Live ISO & Installer

Modern, minimal environment for quick, user-friendly installation.

Our live ISO runs Sway‚Äîa lightweight tiling window manager‚Äîensuring a resource-friendly and responsive installer session. The ISO also includes a Flatpak filesystem and everything needed for thorough testing, making it larger but comprehensive.

Flatpak Integration

All user apps run in isolated containers for consistent updates ü™ê

By distributing user software as Flatpak packages, Shanios ensures apps remain sandboxed from the immutable core. This simplifies updates and preserves system integrity while offering a broad range of software.

Swapfile & Encryption

Built-in hibernation and full disk encryption for data protection.

Shanios includes a dedicated swapfile for hibernation, letting you save sessions easily. Optional full disk encryption secures your data from unauthorized access.

NVIDIA-Open Drivers

Preinstalled support for NVIDIA hardware‚Äîno extra setup needed.

Shanios ships with open-source NVIDIA drivers out of the box, ensuring smooth performance and compatibility on NVIDIA GPUs from day one.

High Performance

Optimized for modern hardware and cloud-based environments.

By leveraging advanced caching, streamlined Btrfs subvolumes, and minimal overhead, Shanios is well-suited for desktops, servers, and cloud VMs where speed and efficiency matter.

Installation

This page will guide you through the Shanios installation process and provide recommendations.

Requirements

  • At least 64 GB of storage
  • 4 GB of RAM
  • A 64-bit processor
  • USB stick with at least 8 GB capacity
  • Graphics card supporting at least OpenGL 3.3

We set these requirements to ensure a smooth experience and enough space for Btrfs snapshots. If your system meets these, you're good to go ü™ê!

Live ISO Credentials

After you boot into the Shanios live environment, if required use:

  • Username: shani
  • Password: shani

Steps to Install

  1. Boot your PC and open your firmware setup (UEFI/BIOS). Disable Fastboot & Secure Boot if possible.
  2. Ensure SATA is set to AHCI mode. If your device supports UEFI, configure UEFI only.
  3. Create a bootable USB using DD, Etcher, or Rufus. Then boot from it.
  4. Start the installer in the live environment. Follow the guided procedure.

Shanios uses a Sway-powered live ISO, offering a modern and minimal environment for installation. After installation, you'll have a minimal GNOME desktop for daily use.

Updates & Flatpak ü™ê

For updating your system, Shanios provides the shani-deploy command. This handles atomic, dual-root upgrades, ensuring you can roll back instantly if needed. Flatpak ü™ê is used for user software, keeping apps containerized and simplifying your update experience.

Recommendations

  • We do not support Dual booting. Another OS might overwrite your bootloader.
  • We recommend real hardware for the best performance. Virtual machines may lack graphics acceleration.
  • Use shani-deploy for reliability. Running raw pacman is not recommeneded.
