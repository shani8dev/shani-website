Immutable Reliability

Atomic Updates That Always Work

Say goodbye to broken updates forever. Every upgrade is atomic and tested before it touches your system.

Shanios uses two complete system copies in an immutable design. While you work on one, updates happen safely on the other. Only after everything is verified working does the atomic switch occur. If anything goes wrong, you're instantly back to your working system.

No more weekend repair sessions. No more lost productivity from botched updates.

Instant Recovery

Something went wrong? Fix it in 30 seconds with a simple reboot. No reinstalling, no data loss.

Traditional systems break when updates fail halfway through. Shanios keeps your previous working system intact and ready. A quick reboot selection brings you back to exactly where you were.

Your files, settings, and apps remain untouched. It's like having a time machine for your computer.

Immutable Core Security

Core system files can't be modified, even by malware. Enhanced security by immutable design.

The heart of Shanios is immutable and read-only. Malware can't touch critical system files because they literally cannot be changed while the system runs.

Your personal data and applications run in separate, protected spaces. Optional full-disk encryption adds another layer of security.

Getting Started

Installing Shanios is straightforward and takes about 20 minutes. Here's everything you need to know.

What You'll Need

  • Storage: At least 64 GB free space (for the dual-system immutable design)
  • Memory: 4 GB RAM minimum
  • Computer: Any 64-bit PC or laptop
  • USB Drive: 8 GB or larger for installation media
  • Graphics: Pretty much any graphics card from the last decade

These requirements ensure smooth operation and enough space for the safety features that make Shanios immutable and reliable.

Live System Access

If you need to log into the live installer environment:

  • Username: shani
  • Password: shani

Installation Steps

  1. Prepare your computer: Turn off Fast Boot and Secure Boot in your computer's settings (BIOS/UEFI)
  2. Set compatibility mode: Switch SATA to AHCI mode, enable UEFI-only if available
  3. Create install media: Use Balena Etcher, Rufus, or dd to write the ISO to USB (avoid Ventoy - it won't boot properly)
  4. Boot and install: Start from USB and follow the guided installer

The installer runs in a clean, modern environment that works on virtually any hardware. After installation, you'll boot into your chosen desktop environment, ready to use.

Keeping Your System Fresh

System updates use the shani-deploy command, which safely updates your immutable system in the background using atomic operations. If anything goes wrong, you can instantly roll back.

Apps install through Flatpak, keeping them separate from your immutable system core for maximum safety and convenience.

Things to Know

  • Dual booting: Not recommended - other operating systems might overwrite the bootloader
  • Virtual machines: Works, but physical hardware gives you the full immutable experience
  • System updates: Always use shani-deploy - don't run pacman directly

The Story Behind Shanios

Every computer user has experienced the frustration of a broken update or corrupted system. Shanios was born from a simple belief: your operating system should be your most reliable tool, not your biggest source of stress.

Created by Shrinivas Vishnu Kumbhar during M.Tech research at Walchand College of Engineering, Sangli, Shanios represents years of experience in building reliable systems. As co-founder of Garuda Linux and a seasoned DevOps engineer, Shrinivas understood that traditional update mechanisms were fundamentally flawed.

The breakthrough came in 2023: what if we could make system updates as safe as turning a light switch? By maintaining two complete system copies and switching between them atomically, Shanios eliminates the possibility of broken updates while keeping the user experience simple and intuitive.

Since our first release in March 2025, Shanios has proven that immutable reliability and ease of use aren't mutually exclusive. We're now growing a community of users who never want to go back to systems that can break.
