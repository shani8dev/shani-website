Why Choose Shanios

A next-generation immutable operating system built on Arch Linux that truly never breaks. Revolutionary blue-green deployment with Btrfs subvolumes ensures atomic updates and instant rollback. The custom shani-deploy tool orchestrates updates with military precision—downloading verified images, applying configurations, generating unified kernel images, and validating everything before switching. Core Architecture - The Foundation

Blue-Green Deployment

Two complete system images (@blue and @green) using Btrfs subvolumes. Updates apply to the inactive image while you work. DevOps-proven strategy adapted for desktop use.

Shanios uses blue-green deployment architecture with two complete system images (@blue and @green) implemented as Btrfs subvolumes. While you use one image, updates install to the other. After verification, a simple reboot switches to the updated image atomically.

The custom shani-deploy tool orchestrates the entire process—downloading verified images, applying configurations via overlays, generating new unified kernel images, updating the bootloader, and validating everything before switching. Updates are applied to the inactive system copy, meaning your current working environment is never at risk.

This dual-image design eliminates mid-update failures entirely. If verification fails, the update never applies. Your active system remains untouched and fully functional throughout the entire update process. This brings datacenter-grade reliability to the desktop.

Intelligent Layering System

Sophisticated architecture balancing immutability and flexibility. Read-only root with smart overlays for configuration persistence.

Immutable system core: Root filesystem (/) mounted read-only, preventing system corruption, malware modifications, and configuration drift while maintaining maximum performance. The system partition ensures absolute stability.

/etc as overlay filesystem: Persistent configuration changes without breaking immutability. /var managed with systemd.volatile=state for optimized state management. User applications via Flatpak for complete isolation. Mutable /home for user data and configurations.

This sophisticated architecture provides the perfect balance of immutability and flexibility, allowing necessary customization while keeping the core system protected and reliable.

Instant Recovery & Zero Downtime

Rollback in under a minute. System updates happen in the background. If anything goes wrong, reboot to instantly return to your previous working state.

Both system images (blue and green) remain intact and bootable at all times. If anything goes wrong after an update, select the previous image from the boot menu. The switch is instant—rollback completes in under a minute, just a reboot away.

Zero downtime updates: System updates happen in the background on the inactive partition. Reboot to switch partitions, and if anything goes wrong, reboot again to instantly return to your previous working state. No interruption to your workflow.

Your personal files, settings, and applications persist across both images using shared storage. Only the system core rolls back, giving you the best of both worlds: instant recovery without losing your work or data.

Complete step-by-step instructions to get Shanios running on your system

Installation Guide

Installing Shanios takes about 20 minutes. Everything you need to know is here.

System Requirements

  • Storage: 32 GB minimum for blue-green dual-image architecture (64 GB recommended)
  • Memory: 4 GB RAM minimum (8 GB recommended)
  • Processor: 64-bit x86 CPU with virtualization support (Intel VT-x or AMD-V)
  • Graphics: Intel, AMD, or NVIDIA (all drivers included)
  • Installation Media: 8 GB or larger USB drive

Note: The blue-green architecture requires two complete system images, which is why minimum storage is higher than typical Linux distributions. This dual-image design is what makes atomic updates and instant recovery possible.

Pre-Installation: BIOS/UEFI Setup

Configure firmware before installing (access via F2, F10, Del, or Esc at startup):

  • Disable Fast Boot
  • Disable Secure Boot (required for installation—can be re-enabled after enrolling Shanios keys)
  • Set SATA mode to AHCI
  • Enable UEFI boot (disable legacy/CSM)
  • Enable virtualization (Intel VT-x or AMD-V)

Note: If you want to use Secure Boot, you can re-enable it after installation and enroll the Shanios signing key using MOK (Machine Owner Key) management.

Creating Installation Media

Write ISO to USB using Balena Etcher (recommended, cross-platform), Rufus (Windows), or dd (Linux/Mac).

Important: Do not use Ventoy—it has known compatibility issues with Shanios's immutable architecture and bootloader configuration.

Installation Steps

  1. Boot from USB (press F12, F2, or Del during startup)
  2. Select "Install Shanios" from boot menu
  3. Choose language, timezone, and keyboard layout
  4. Select disk and partitioning scheme (automatic recommended)
  5. Enable LUKS2 full-disk encryption if desired (recommended for laptops)
  6. Wait for installation to complete (10-15 minutes)
  7. Reboot when prompted (remove USB drive)

Note: Desktop environment (GNOME or KDE Plasma) is determined by which ISO you downloaded.

Initial Setup (First Boot)

On first boot, the Initial Setup wizard will guide you through:

  • Creating your user account (username and password)
  • Configuring network connections
  • Setting up privacy and location services
  • Finalizing system preferences

This approach allows the base system to install quickly while personalizing your setup on first boot.

System Updates

Immutable system updates:

sudo shani-deploy

Updates apply atomically to the inactive image. After reboot, you boot into the updated system. If needed, you can select the previous image from the boot menu.

Application updates:

flatpak update

Firmware updates:

fwupdmgr update

Delivered via LVFS using fwupd.

Important: Always use shani-deploy for system updates to preserve immutability.

Important Notes

  • Dual booting: Not recommended—other OSes may break the bootloader
  • Package management: Use Flatpak, containers, or AppImages
  • System modifications: Root filesystem is immutable—use proper channels
  • Secure Boot: Can be enabled after installation by enrolling MOK keys (password: shanios)
